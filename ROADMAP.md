# 🚀 專案路線圖 (Roadmap)

本專案旨在成為一個強大的 RAG 實驗平台，未來計畫朝向以下方向發展：

## 第一階段：增強 RAG 實驗能力 (短期)
- [x] **多模型並列回答**：支援同時比較不同 LLM 模型 (Gemini 3.1 Flash vs Pro) 的回答。
- [x] **自定義 System Prompt**：允許使用者調整 RAG 回答時的系統指令。
- [ ] **混合搜尋 (Hybrid Search)**：結合向量搜尋與關鍵字搜尋 (BM25)。
- [x] **Re-ranking 整合**：已實作 LLM Re-rank 功能提升精準度。

## 第二階段：視覺化與分析 (中期)
- [ ] **向量空間視覺化**：使用 D3.js 或 Three.js 呈現 Chunk 在向量空間中的分佈。
- [ ] **相似度熱點圖**：視覺化呈現 Query 與各個 Chunk 的相關度分佈。
- [x] **回答品質評估 (RAGAS)**：已實作自動化評估指標（忠實度、相關度）。

## 第三階段：企業級功能 (長期)
- [ ] **多文件支援**：支援同時對多個檔案進行跨文件檢索。
- [x] **對話歷史 (Chat History)**：支援多輪對話，並考慮對話上下文的 RAG。
- [x] **匯出報告**：一鍵匯出比較結果為 PDF 或 JSON 報告。
- [ ] **API 支援**：支援更多的 Embedding 供應商 (OpenAI, HuggingFace, Local Models)。

---

## 💡 開發建議
- **效能優化**：當 Chunk 數量過多時，考慮使用 Web Worker 進行相似度計算，避免阻塞 UI。
- **UI/UX**：增加「一鍵複製」回答內容的功能，方便使用者整理實驗結果。
- **安全性**：加入 API Key 管理介面，讓使用者能動態切換不同的 Key。
